Blueberry Group : New Issues / Tasks / Tickets Management Pages 

Created by Jason Cozza on Nov 24, 2020

Introduction

I think we need to do a big overhaul of tasks/issues management inside PTS - it is far too slow, and is  wasting a lot of people's time.
 

  1. I think we should have merged issue+tickets+tasks page, so a PM can look through all items that need action on a project. 

     
  2. Users should be able to see a list of issues/tasks in an LHS panel, and then open the details in a right side panel - this should be much faster than switching to a new screen.  Obviously it can be difficult to get everything on one screen, but wide-screens are common. 
     
  3. We are potentialy standardising on Office 365, and maybe we could integrate with it more directly - this might make it easier to discuss PTS issues in outlook?

Review / Edit Issues without Opening New Window

I think the best idea is to have a dual mode screen.  I open up the issues/tasks/tickets search screen, and I  can perform filtered searches and see a big grid.  I can use a tooltip to view more details.   If I click the issue
title, the grid shrinks to only a few columns, and an editor panel appears on the right.  I can now click on issues and they appear in the right hand side panel.

Another option:  we display the detail window in a separate screen.  Some BB people spend a lot of time reviewing issues, and they could have one smaller screen to show the issue list, another larger screen
to show the detail.   To support this we'd need to have a special mode which can be enabled and saved to that PCs' cookies. 

Review Issues / Tasks / Tickets in One Screen

I think it's useful to have a single screen where I can see all issues and tasks for a project, and  review / close them.  The "issue editor" panel would change depending on which item is selected.

I'm not sure what issues this presents?   I guess for issues / tasks do not have identical status values which may make filtering more complex?

Implementation Approach

I suggest

  • We review and improve the new Angular issues screen, taking time to get it right - and thinking of it as a template for other screens to come
     
  • We implement the dual mode with a dummy issue review panel, and see how this feels
     
  • We then work on the full issue panel, so we can put this into general use
     
  • The we do tasks screen and tasks panel
     
  • Then we create a merged tasks / issues screen
