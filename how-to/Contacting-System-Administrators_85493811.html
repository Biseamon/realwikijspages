Blueberry Group : Contacting System Administrators

Created by Jason Cozza on Nov 24, 2020

For urgent/critical issues or tasks, contact sysadmins via email at sysadmins@bbconsult.co.uk.

A non-exhaustive list of things which sysadmins consider urgent:

  • System failure (production system: database, infrastructure, server, etc.).
  • Overdue task.

If your task does not fit into the above, continue with this guide.

Be reasonable about task deadlines. For example, if you've decided to submit a task on the day that it needs to be completed when it could have been submitted a week prior, please don't be offended if a sysadmin says that it can't be prioritised. Simply setting something as urgent in PTS won't make it move any quicker, and if it's truly urgent you should be using email.

Sysadmins appreciate that customers can make things difficult with regards to scheduling, but do try to leave as much time for completion as possible.

Do consider whether the task you are submitting is actually for sysadmins. For example, if it's an issue with PTS you should contact Carl Smith or Oliver Goodman.

Step-by-step guide

  1. Open PTS
  2. Click the Contact Sys Admins button at the top-right corner of any page.
  3. Fill out the form as desired.
    If your work is project specific, then remember to select it.
    Try to include as much information as possible in the Further supporting information field to reduce the time taken for work to begin on the task.
    You can suggest who should take up the task by mentioning them in the Further supporting information field.
  4. Click Send.


Alternatively, you can create a task/issue/ticket as normal in PTS and assign the Sys Admins group to ensure they receive it.

Key points

Sys admins will automatically be informed of the new task and will delegate it internally.

When they want you to respond to the task, they will set the status to Waiting for End-User.

When you've responded and want them to respond or continue work, set the status to Waiting for Manager/Tester or Waiting for Developer team.

Respond to tasks as quickly as possible so things can move quicker.

Don't split discussion of a PTS task into email unless it has evolved into an urgent task. Sysadmins will simply direct you to use the PTS task if you try this.

If you feel that your task has been ignored (no responsible set, or no initial follow up), do send out a reminder to sysadmins@bbconsult.co.uk.

Try to keep the task on-topic. If the task drifts away from the original topic and the task requirements were fulfilled, sysadmins will likely close it as completed so you should create a new task with new requirements if required.
