Blueberry Group : Create Ticket

Created by Jason Cozza on Nov 24, 2020

This feature lets you quickly create a ticket on the PTS using its ability to read special code in emails to log a ticket.


Why an add-in to create a ticket ?

As you may know, we can already create tickets either using the website or sending an email to a dedicated email-address with some tags in it.
The PTS has become a bit slower recently as there are always more data on it. For the support team, logging any new incoming ticket has not only become longer but unpleasant.

We have always been aware of this possibility to create tickets sending that kind of email : 

"
To:  ticket-blu-ptssupport@pts.bbconsult.co.uk
-----------------------------------------------------------
##auto-create-issue
##subject PTS Slowing Down
##report-by support@blueberrysystems.co.uk
##Status Open
##priority High

The PTS is becoming bigger and bigger. Not its fault, but how to work more efficiently now ?
"

But as you can see, logging a simple sentence needs 5 more lines including a specific email address based on the item you want to create, the customer code and the project name ...
Not to mention that if you want to log a ticket with a picture you first have to create the ticket, update it then go in attachments and add it. If you want to create an issue linked with the ticket, you then have to click the corresponding button.
Doing all this in three steps may also be unpleasant, moreover you will be waiting for the website to load between each step. Or you can do all at once with a "simple" email ! 

Fortunately, that is where this add-in is incoming, it will help you generating this email for you with a simple interface and let you log any ticket within seconds.


How to do ?

This feature is loaded as a button in the ribbon tab Blueberry in group PTS Helper.


PTS Helper group and Create Ticket button.

You understand that creating a ticket is based on any customer email. That is why you have to select an email first, otherwise you may see these error messages popping :


Selection error messages after clicking "Create Ticket".

NB: You can also open this window clicking the button on an open email which will directly open this window for the current email.
       It comes very handy when used with the Watchdog opening the unanswered emails ... 


Opened email screen with "Create Ticket" button in the BLUEBERRY tab.


Once this button is clicked, this simple window will appear :


Window "Create Ticket" after clicking the related button.

This window is detailed in five different parts as following :

  • Subject : The subject of the PTS ticket you want to create.
  • PTS Project : The PTS Project in which you want to log the ticket.
  • Auto-create issue : Whether or not you want to create an issue using the details you provided for the ticket's creation.
  • Status : Open or close. Mainly there in case we may select statuses like WfDt in the future.
  • Priority : Represents the priority of your ticket. You can choose between later, low, normal, high and urgent priority.


A few things you want to know

The add-in auto-completes the subject field with the subject of the email. Feel free to edit it at your convenience.

The project list is updated at Outlook's startup using a webservice on the PTS (Thanks Oliver !). To avoid to load the PTS unecessarily, it is not re-updated each time the window is loaded but you may want to refresh the list.
To do so, simply click the button on the right of the list. It will use the webservice to update the list.
This list only contains projects of which status is "In Support" on PTS. Why would logging a ticket for a project that is not ?

The bottom part is loaded with auto-create issue ticked, open status and normal priority by default when opening the window.
The top part has the subject copied from the email's one.

A piece of code tries to match a pattern between any email address in the To and CC fields (for example: "bims-support@bbconsult.co.uk; Anne-Marie amf <anne-marie.fallon-cotton@bbconsult.co.uk>; Michelle Finn <michelle.finn@bbconsult.co.uk>; Jon Baker <jon.baker@bbconsult.co.uk>; Jason Cozza <jason.cozza@blueberrysystems.co.uk>") and the project list.
Therefore, the correct project may already be selected. On this screen, the code has matched "bims-support@blueberrysystems.co.uk" with project "BIMS Support".

You understand that the most project that there will be in the list, the less chances you will have to get a good match as the code is pretty simple.


One step further

After fulfilling all these information, you can click OK and a new email window will pop with generated text in it :


New email window containing generated text after clicking "OK".

You can then notice a few things :

  • The email is in plain text. It is normal as we just want text as the description of the ticket, no need for more.
  • The diverse documents that were in it are auto-attached here. Feel free to add/delete any of it.
  • The generated text is added above a forward-like email. You may want to erase all the details like From, To, CC fields [...]

At this point, you may want to modify the subject of the ticket. If you do so, try to match the subject of the mail and the subject of the tag ##subject for coherence (You will also understand why in the following).

The code that is attaching elements to the email is saving every single enclosed element within a folder and attaching them to this new email. Therefore, if you receive an email with a picture made as layers, you may experience issues. Also, if anyone in the conversation is using pictures for their signature, it may be enclosed as well. Do not forget to check the attached files before sending this email as it will then be part of PTS ticket's attachments.


All done ! Now just click Send and everything will be computer work, whatever the time it takes to be created, whatever the attachments, if you want to create an issue or not [...] everything will be done and you will not have to wait for anything.


Auto-Answer to Customer

As if it was not already enough, there is one left thing that this feature will be doing for you. It will track the answer email from PTS that should look like this one :


Answer email from the PTS once the ticket has been created.

You may have noticed that this email contains a very valuable information that we want to get : The ticket number.
You may also notice that this email has been sent by noreply@pts.bbconsult.co.uk and that its subject is "RE: <ticket_subject>".

This is exactly what the add-in will be waiting for, this precious email from the PTS that informs you of your ticket number. It will be looking for an email sent by PTS email address with your former email subject.
That is why it is important that your email subject and tag ##subject are the same, to avoid any misunderstanding and to make things easier for you.

Once this email has been detected and the information extracted, a new email window should pop on your screen which is a Reply to All with an easy answer containing the ticket number.
