SQL Standard vs. Web Edition

In some cases Express Edition is not enough, either because of the 10GB database size limit, or the CPU/RAM limitations (1CPU or 4 cores and ~1.4GB RAM), or both.

Web Edition is the best alternative in this case for production use (see below about the Developer edition), as it's quite cheaper than the Standard edition and has the following features:

  • database size up to 524 PB
  • CPU: use 4 sockets (hardware) or 16 cores - the one which is less
  • RAM: 64GB
  • SQL Agent is enabled

Here are the main limitations of the Web Edition comparing to the Stanrard one, in case some of them are critical for some project:
(please also see the original here)

  • using only 16 CPU-cores instead of 24 cores in Standard
  • using only 64GB RAM, instead of 128GB in Standard
  • no compressed and encrypted backups
  • limited clustering support
  • acting as a "subscriber" only in the replication
  • web edition doesn't support SQL Server Profiler UI,  If you point a Standard Edition SS Management Studio at a Web Edition database engine, you’ll get profiling. If you point a Web Edition SS Management Studio at the same database engine, you will find no profiling UI.

SQL Developer Edition

Please also consider using the Developer edition on the TEST-servers, for development process, etc.
(https://docs.microsoft.com/en-us/sql/sql-server/editions-and-components-of-sql-server-2017?view=sql-server-2017#includessnoversionincludesssnoversion-mdmd-editions)

As noted on that page:

SQL Server Developer edition lets developers build any kind of application on top of SQL Server.
It includes all the functionality of Enterprise edition, but is licensed for use as a development and test system, not as a production server.
SQL Server Developer is an ideal choice for people who build and test applications.

So, it's:

  • equal to Enterprise edition
  • is free
  • is not an "evaluation" copy
  • is not for the production env., but for development purposes only
