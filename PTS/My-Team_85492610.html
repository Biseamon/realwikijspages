My Team 

The My Team screen allows Managers to define which developers they relevant to them. The benefit of this is that it makes it easier for check the availability of those particular developers. For example, in the weekly holiday summary, team members are listed separately from all other BB staff to make it easier to find them.

For developers the My Team screen allows them to quickly check who their Project Managers and Assistant Project Managers are and to also see which other developers are working on the same projects as them.

Managers

When visiting the My Team page. a manager will see the following:


The first box contains any developers who are part a team that the manager is project manager or assistant project manager for. These team members can be updated by going to the Project Team tab on the Edit Project page for a specific page project (see below). Any changes made on the Project Team tab will be inherited by the My Team page.

 

 

Developers who are managed by a manager but aren't part of a project team can be added using the second box. When the Add button is clicked, a dialog will appear containing all BB employees - clicking the + sign next to any of them will add them. This is likely to be used for non-developers, such as testers or support, who are not part of a project team but who a manager still has responsibility for.

Finally, the third box allows a manager to add developers who are not managed directly by the manager but who are still relevant to them. As with the previous box, he Add button is clicked, a dialog will appear containing all BB employees - clicking the + sign next to any of them will add them.

Developers

When a developer visits the My Team page, they will see the following:

 

The list of team members consists of any managers or developers who are on the same project teams, as well as any managers who have manually added the developer to their team. Project managers and assistant project managers are indicated by 'PM' and 'APM' next to their name.
