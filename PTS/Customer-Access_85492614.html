Customer Access 

Some times it may be helpful to give a customer limited to access to PTS - this would generally be so that they can get visibility of outstanding tickets / issues or something similar. This achieved by assign roles to the user to grant them the appropriate permissions in PTS.

There are currently several roles that could potentially be assigned to a customer user:

  • Simple Customer: The most basic role - the user can simply log in and view / edit tickets
  • Customer Portal: The standard role for a customer - when logged in, the user sees the customer portal, which includes various details of the project. The user can also view tasks / issues, contact the PM and receive notification emails
  • Extended Customer: The most advanced role - the user can view / edit tickets, issues and tasks, as well as being able to contact the PM

There are also a couple of slightly more specific support user roles available:

  • Customer Support User: The user receives acknowledgement and updates emails and can edit tickets
  • Hubbell Support User: Similar to the standard Customer Support Role but the user also receives response from PTS and is automatically added to project teams

To give a customer a role:

  1. Go to Admin -> Manage Users
  2. Either add a new user or search for an existing user
  3. Click edit
  4. Go to the Groups, Roles and Permissions tab (if a new user was created, set a password on the Basic Details tab first and click Save)
  5. Select the relevant role in the Roles Added To This User drop down and click Add
    Note - if a suitable role is not available, contact Oliver Goodman and he will create a new role that is suitable for the user.
  6. Click Save
