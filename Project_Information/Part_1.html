Discount all users from MS SQL Server DB 

 

Use Master
Go

Declare @dbname sysname

Set @dbname = 'databaseName'

Declare @spid int
Select @spid = min(spid) from master.dbo.sysprocesses
where dbid = db_id(@dbname)
While @spid Is Not Null
Begin
Execute ('Kill ' + @spid)
Select @spid = min(spid) from master.dbo.sysprocesses
where dbid = db_id(@dbname) and spid > @spid
End

Watchdog 

 

 

A Watchdog - What is that? 

 

It is a simple feature that checks that certain emails are answered within a preset delay. If not, a notification (As a MessageBox) pops and warns you about any unanswered email.
 

Here's a screenshot of a popup generated by Blueberry Outloot.
 


If you click "Open" it opens the email so you can review and reply to it quickly if necessary.


Here are a few things that you have to know :

  • As it is an add-in, it may slightly slow down Outlook's startup.
  • On startup, the add-in watches for all the unanswered mail within the max delay you specified in the settings.
    So if you have just started your working day, you instantly know if anything requires your attention.
  • The add-in creates a new tab to Outlook's homepage called "BLUEBERRY" which contains a group named "Watchdog" : there is one toggle button that enables/disables the add-in and one button to change its settings.


How it works :

The add-in looks through all the emails received within the delay you specified.
In these last emails, it checks that they are not blacklisted (Or that they are white-listed depending on your settings). Once done, it checks that the To or CC field contains at least one string specified in your settings.
If all those steps are validated then a timer starts and if it comes to the end, the MessageBox pops.
All along these steps, the add-in checks if the watched emails have been answered. As in Blueberry we always reply to all, the address that was contacted will also receive the answer and so the timer of the email will be stopped as soon as the add-in has detected the answer.

NB: You understand that an email is considered answered when any email that you receive is more recent and the former sender is now in To or CC field.
       There is no check about the purpose of the email.


How to manage the Settings :

If you want to know more about the Settings, please see below

 

Settings Screen 

 

This is what the Watchdog's Settings screen looks like :


Blueberry Outloot - Watchdog Settings Screen

First, as the plugin has originally been released for the support team, you can see options like "Support Hours" and "Support Days". Do not worry, you can freely use this functionality without doing any support, here are all the explanations you may need about the different settings.

The Settings screen can be decomposed in 4 parts. I will do so to explain you each single setting of it.
 

1) Popup Delays :

The Popup delay part contains two numeric fields :

 


  Settings Screen - Popup Delay Fields

Popup delay represents the delay you allow to any email to be unanswered.
Given this previous screenshot, the emails have up to 15 minutes to be answered. If no answer is received within this delay, a popup will appear to remind you to answer the email.

Popup max delay stands for the maximum delay within which you want to be warned about any unanswered email.
Basically, this setting should be useful only once a day since you are notified after 15 minutes. But in the case you just arrived at work, you may want to know what emails are critical within a larger amount of time.
Given this previous screenshot, arriving at work at 8:30 and starting Outlook will let me know if an email has not been answered since 8 o'clock.

NB: If you disable then re-enable the watchdog, you'll be notified of all the emails unanswered within the last 30 minutes with this settings.
       It is the same if you close re-open Outlook.


2) Support Time :

The Support Time is made of the Support Hours and the Support Days :


                              Settings Screen - Support Hours & Days

Support Hours contains two masked fields :

  • Start which corresponds to the starting watching time.
  • Stop which corresponds to the end watching time.

For instance, if I start Outlook at 6 o'clock and a mail is unanswered, I will not be notified of it. The watchdog will wake up after 8 o'clock.

Support Days contains 7 check-boxes corresponding to the 7 days of the week.
Again, if there are unanswered emails in days that are not ticked, you won't receive any notification.

These previous settings mean the watchdog will warn for any unanswered email between 8 am and 5:30 pm on working days only.


3) Filter List :

The Filter List is composed of many controls :


Settings Screen - Filter List

You can notice a check-box next to "Enable" that let you enable/disable this setting whether you want (or not) to set a blacklist/white-list.
You then have a combo-box  that let you choose between a blacklist and a white-list. It is up to you if you prefer to be warned on a few email addresses or on any address except a few.
A text-box that let you type anything to add to the list. You then have to press the + button to add it to the list (If not already in).
The - button removes the selected items of the list. You can select many and delete them all at once.
A list-box containing all the strings that you want to filter.

NB: This list is case-insensitive, no need to add noreply, NoReply, nORepLY [...]
       Furthermore, it does not work on a full word match principle. Which means you can add "noreply" to stand for every single email address that would contain "noreply" (noreply@bbconsult.co.uk, noreply@google.com, noreply@confluence.bbconsult.co.uk, etc...).


4) Strings to look for :

This list is almost like the previous one :


Settings Screen - Strings to look for

The content of the list-box is the strings that should appear in the TO or CC fields of your received emails. As soon as one single item of this list is found in one of these fields, the email is watchable, otherwise it is filtered.
As the add-in was originally designed for the support team, this list contains "support". It means that any email sent to any support email list (coloplast-support@blueberrysystems.co.uk, vtg-support@blueberrysystems.co.uk, blueberry-support@bbconsult.co.uk, etc...) is a valid one.
Here, if an email is sent to every single person of the support team without the support mailing list in To or CC field, the add-in will not warn about the email if it turns to be unanswered.

Warning :

This filtering is based on the display names in To or CC fields. It means that if you specify something like "martin.green@bbconsult.co.uk", the add-in may not work if the Sender has the email address in its contact with a full name like "Martin Green".
Considering this, I would advise you to put just "martin" in the list in this example.

NB: This list is also case-insensitive, no need to specify all the possibilities like Martin, martin, mArTIn [...]
       It is not working on a full word match principle either. Which means that "support" will stand for any entry containing "support" (coloplast-support@blueberrysystems.co.uk, Coloplast Support, insupportable@gmail.com, etc...).

 

Saving the Settings :

Once you have entered your desired settings, you can either click the "Cancel" button to quit without saving anything or click "OK" to save it all.
A verification of the different impossible cases is made before saving, for example, your max popup delay can not be less or equal to the popup delay.
The Settings Screen let you type any nonsense thing but will catch you up before saving.

There you can either read the error message and correct your mistake(s) or just click "Cancel" to leave without saving. 

Once the settings saved (After clicking "Ok") the watchdog cleans every former watched emails and immediately restarts one (if Enabled). Therefore you may directly have some popups after saving and do not forget the first scan will be based on the max delay in this case.

Blueberry Sleep Monitor Tool 

 

This is a tool for monitoring and logging machine inactivity. It runs as a background task and sends usage stats off to a server which can be used to determine the amount of inactivity for a particular machine. It could for example be used to determine which machines should be turned off which would help to save power.

 

 

 

Setup

  1. Open the web app in a browser. https://sleepmon.bbconsult.co.uk/
  2. Click the sign in button on the top-right corner.

 

3. Sign in with your Google account.

4. If this is the first time you use this account, a pop-up will inform you that a new account was created.

Creating the domain

  1. Navigate to the My domains page. https://sleepmon.bbconsult.co.uk/domains
  2. Click the Create a new domain button.


 

3. Enter a name for your domain and click the Create domain button.

4. You will then be redirected to the domain__s page. It currently has no users.

 


 

5. Click the Copy domain ID to clipboard button to be able to paste the ID later.

Adding users

  1. Navigate to the Download page.
    https://sleepmon.bbconsult.co.uk/download
  2. Click the Download installer button to start the download.

3. Open the installer and click the More info link to allow the installation.

4. Click Run anyway to start the installation.

 

 

5. Click next until you are asked to enter the domain ID, then paste the ID previously copied in the field.

 

6. Click Install to confirm the installation.

7. Sleep Monitor is automatically started when the installation is finished.

8. More info about installation can be found on the download page. https://sleepmon.bbconsult.co.uk/download

 

 

Note that Sleep Monitor runs in the System Tray and isn’t started using a desktop shortcut. It will run automatically upon startup.

 

 

You can right click on the icon in the System Tray to either close Sleep Monitor or change the Domain ID.

 

 

Checking usage

  1. Navigate back to the domain page.

Developer Tasks - Mini Project Features 

 

Mini Project Concept

 

Sysadmins don’t use PTS, nor does Marketing.  They find it too clunky and complex.  A lot of this is because PTS is slow - if we fix issue editor screens, we can improve this.

But Sysadmins and Marketing have other issues – they have lots of mini projects, for example:

  • Migrate Exchange to 365
  • Fix LDAP Security

The problem is that creating each one of these as a PTS project requires loads of fields that  aren't required.  And these mini projects are too small to really get their own project ID.

Really we need a concept of a mini project.

My thinking is that we should make it so that individual tasks can be Mini Projects – we promote a task to be a mini project, and then maybe some extra fields appear.

We can then add extra screens to PTS to show all the mini projects by some criteria.

Question:  can we have mini projects at any point in the tree?   I don’t know.  If we allow Mini Projects to exist at lower tree levels, then we can use the hierarchy to organise them - so we can have a collection of security tasks.  But this does make it a lot more complex.

I'd probably say that mini projects can't contain other mini projects, and any folders above a mini project must be just a container.  But it needs more thought

Mini Projects would exist within a normal PTS project, so we might have a PTS project for Sysadmin, one for PTS, and one for BBWT - each including a list of significant mini projects.

Project Groups

Are mini projects another way of doing project groups?   At present, a PTS project is completely standalone.  This quite often gives us a problem:  if we get two related projects from customer - which share code and timescales - it feels wrong to put them into two different projects.  

Maybe Mini Projects could fix this - we could have a project called BIMS with mini or subprojects of EOBS and Core?

 

What do we do with Mini Projects

 

The main thing that mini projects allows is for us to make a list of all current mini projects, perhaps with an owner, and some idea of time left or completion date

I suppose Mini Projects would also gain other features of big projects, but I'm not sure exactly which yet 

One idea is to publish mini project details to the wiki?

Dependencies / Multiple Parents

PTS at present has a strict hierarchy which is currently used to model the functional breakdown of tasks and dependency.  This is a bit limited compared to other tools
 

This becomes more important when using PTS for syadmin tasks, because it's more likely there will be common dependencies between mini projects.  If two projects are dependent on one common task, we can't represent this in PTS properly. 

The big question is how exactly to present this in the UI?  UIs to represent tree structures that support multiple parents are very hard to do well - users struggle to know how to delete one parent, for example.

Options:
 

  1. We have a simple system:  a mini project can be linked to multiple tasks elsewhere in the containing project as dependent tasks, but this isn't in the tree as such
     
  2. We allow a task from another mini project to be dragged into the tree and it appears with a shortcut icon.  Note that marking the imported task in this way means that each task actually has one real parent and inbound links from other places.  This may simplify the logic a bit.
     
  3. Maybe we only allow mini projects to be dependent on other mini projects.  This feels too limited.

Amazon Web Services: Deploying a new live version of AWS BVI 

Checking Lambda:

Before deploying a new live version, the lambdas need the following checks:

  • Cognito_BVI_Post_Authentication_Python
    • Check if every SQL query are functional.
  • Cognito_BVI_Pre_Sign_up_Python
    • Check if every SQL query are functional.

Note: Grabcut and BVI engine are also using SQL so they need a check as well.

Test Run:

Once all lambdas have been checked and change to use the next live values. A full test is required and any critical fail need fixing.

For this it is best to double check that all API Gateway end points are using the correct Lambda on the beta stage.

Deploying a new live version:

Once the future version has passed the test program, create a new version of the lambdas and stage using the instruction at the following link: https://wiki.bbconsult.co.uk/display/BLUEB/Amazon+Web+Services%3A+Deploying+a+stable+stage+of+an+API+gateway+calling+Lambda

The new stage should be named “app_test_” followed by the release date. Ex: ‘app_test_17_02_17’.

Once the API deployed reverse all method to use the latest lambda to avoid having to do so when deploying the next time to beta.

Then make a copy of the swagger file and rename it to contain the new stage name.

The next step is to update the app_test usage Plan:

  • Go to Usage Plans
  • Select the usage plan you want to add to stage to.
  • Under "Associated API Stages" choose the BVI Web Service API for the API and then choose the new created Stage.

Amazon Web Services: Create a new versioned Lambda Function

  • First go to the Function console https://console.aws.amazon.com/lambda/home?region=us-east-1#/functions?display=list and click on the function name you want to create a new version of.
  • On the action menu select Publish new version a pop up will asking to enter a description of the version.

 

  • Press the publish button once you added some useful text to the description.
  • Once publish you will be sent on the version page of the lambda. Although it says you are now viewing the code of new version it is not the case this is why given a good description of the version is useful to recognize it.
    Note that the Qualifiers button change to Version: [id]

 

  • Switching from one version to another can be done by pressing  the “Version : [id]” / “Qualifiers” button and selecting the version id on the versions tab.

Amazon Web Services: Call lambda on API Gateway by its version

  • Go to the AWS API Gateway console and select your API.
  • Under resource select the method you want to change lambda version.
  • Go to the Integration Request.

 

  • Click on the Pen next to “Lambda Function” and add “:” + the version id to the function name.
  • Press the check mark to save the change.
  • Repeat process for all method you’d like to use lambda function.

 

  • Then deploy the api to an existing stage or create a new stage using the action button, Deploy API option.

The method will now call the versioned lambda instead of the latest lambda.

To get started with using AWS Lambda this link will get you through the basic tutorial of making a lambda function: http://docs.aws.amazon.com/lambda/latest/dg/getting-started.html

Boto3:

Boto3 is the AWS SDK for Python. Its documentation can be found here: https://boto3.readthedocs.io/en/latest/

To use Boto3 on AWS lambda you just need to import the module in the code as amazon already has it installed. You will also need to give your lambda a role which allow access to the resources your function will be using.

To use Boto3 on a computer you will need to install boto3 on your python environment and save a IAM key in a config file in your user folder as explained here:

https://boto3.readthedocs.io/en/latest/guide/quickstart.html#configuration

For example, the location where you will save the key in windows would be: C:\Users\User-Name\.aws

 

To learn how to access AWS resources Boto3 has a good documentation here: https://boto3.readthedocs.io/en/latest/reference/services/index.html

There are two way to access to Aws resources:

  • By using the client: client = boto3.client('sqs')
  • By using the resource: sqs = boto3.resource('sqs')

The main difference is if you need to create new resources like a new SQS queues you will need to use the client and if you just need to access existing resources using ‘resource’ is better although you can also do it using the client.

Import Package AWS don’t have by default:

To import package that AWS doesn’t have you will need to create a zip file with the package in it and your python code. Then upload your code to AWS using the upload from a zip file option and in the configuration of your lambda change the Handler to: ‘file_name.method’ where file name is the name of your python file and method is the name of your event handler method.

To know what exactly needs to be added to the zip file read here: http://docs.aws.amazon.com/lambda/latest/dg/lambda-python-how-to-create-deployment-package.html

Also, please note that AWS run with Linux so some package installed on windows might be missing stuff to run on Linux environment like it’s the case with ‘pymssql’ so it’s best to add package installed from a Linux environment.

How it’s set up:

Attributes

Once a Cognito user poll has been created you cannot change the required attributes from the list of proposed attributes. Nor can you delete a custom attribute once created.

To make a custom attribute required you will need to create a lambda that gets triggered at post register (see Triggers below).

The only attributes used are Email and custom:API_Key all other attributes are legacies from an old plan/setup.

Policies

Password minimum Length is 8, requires: Numbers, Uppercase letters and Lowercase letters.

Users are allowed to sign themselves up (via email validation).

After 7 days if a created account hasn’t been used, it should expire.

Verifications

Verification is made by email and MFA is disabled.

Message customizations

Message customization is handled by using the custom message trigger. The email used is bvi-support@bbconsult.co.uk

Devices

Device remembering is set to NO because we couldn’t make the .Net SDK log back in with the remembered device. See: Cognito for .Net

Apps

The currently used app is mybvi2 which has app client secret enabled as well as admin_no_srp_auth and its refresh token expires in 30 days.

Triggers

Documentation about Cognito triggers can be found Here.

Our Cognito uses three triggers:

CustomMessage:

Used for customising the email, sent on a new register, or on forgot password.

Example: aws-lambda-triggers-custom-message-example

Documentation: cognito-user-pools-lambda-trigger-syntax-custom-message

PostAuthentication:

Check if user exist in DB and if not, or if values of AppApiKey, SubscriptionType or ScansRemaining are null, It creates/updates the user.

Example: aws-lambda-triggers-post-authentication-example

Documentation: cognito-user-pools-lambda-trigger-syntax-post-auth

PreSign-up:

Used to check if the given API_Key in custom:API_Key exist in the DB.

Example: aws-lambda-triggers-pre-registration-example

Documentation: cognito-user-pools-lambda-trigger-syntax-pre-signup

 

No particular extra permission are needed for the lambda that will get call by Cognito.

 

Cognito for .Net: 

 

Using Cognito in .Net is a bit tricky as it doesn’t have all the call tidied up like the other SDKs do to Authenticate, register, validate…

Instead you need to handle all of the crypto and making the SRP protocol work. Thankfully a feature request has been opened and some people shared their solution here: https://github.com/aws/aws-sdk-net/issues/495

Those solutions are mostly based on the JS SDK: https://github.com/aws/amazon-cognito-identity-js/tree/master/src

In the end this is what has been used in the BVI Test app: https://gist.github.com/BBNicolas/5ab3f1f8c604ebe2e4ecf14c35487495

-          Validation

-          Login

-          Forgot password

-          Refresh

-          Logout

Cognito Verify Token 

 

Using a token with AWS API Gateway is really easy, you can find a step by step instruction in AWS documentation here: http://docs.aws.amazon.com/apigateway/latest/developerguide/apigateway-integrate-with-cognito.html

Create a new API or select an existing API in API Gateway.

From the main navigation pane, choose Authorizers under the specified API.

Under Authorizers, choose Create and then choose Cognito User Pool Authorizer.

To configure this authorizer:

Choose a region for Cognito region.

For Cognito User Pool, choose an available user pool.

The Authorizer name field will be automatically populated with the chosen user pool name. However, you can customize it if you want to.

The Identity token source field will be set to method.request.header.Authorization by default. However, you can customize it if you want to. Using the default, Authorization will be the name of the incoming request header to contain an API caller's identity token.

Optionally, type a regular expression in the App client ID regex field to validate client IDs associated with the user pool.

Choose Create to finish integrating the user pool with the API.

Having created the authorizer, you can, optionally, test it by supplying an identity token provisioned from the user pool.

 

Doing the verification yourself is a bit more work but a aws blog show how to do it:

https://aws.amazon.com/blogs/mobile/integrating-amazon-cognito-user-pools-with-api-gateway/

Shape GB 

 

Contact

 

Basic Description

Customer has an app that will take two pictures, one from the front, one from the side, create an outline around their profiles, and then send the outlines to the server along with the metrics (height/weight/ethnicity/gender), via a scan/startsgb post call.

They get a scan ID returned, and then will send in a JSON packet which contains the results of the survey taken before the images were taken.

The Survey will be linked to the scan via this scan ID.

 

Examining the data

The credentials are in Bitwarden, under BVI ShapeGB SQL Express Server. Use SSMS to log in.

The most important tables are Job, SgbSurveyReponses, and the view JobMeasurements.

dbo.Job

The following are important columns

Status

  • This shows the status of the scan. If it is a 6, it means that all stages are fine, as there are six stages, and once each stage is passed successfully, it will increment this status.
  • -1 means we have detected an issue, normally outside of the system.
  • any number lower than 6, means it has failed on the next stage (so a 5 means that there was an error in stage 6, when processing the scan)

Error Text

If the scan failed (specifically if it was -1), the contents of this should give insight into what has gone wrong.

Height/Weight/Age/Gender/Ethnicity

These are all survey information entered by the app.

 

SgbSurveryResponses

This table stores the survey responses, taken through the app, in a JSON format. Each survey is linked to to a Job.ID

Example Response:

 

JobMeasurements

This is a view created from two tables:

OrdMeasurementsKind which contains the index and measurement area. There are different apps that handle scans, so there isa  set of indexes for various measurements per app. For example, the latest app uses ID 297-312 as it's indexes for measurements.

OrdMeasurements which fills in the values attributed to the indexes found in OrdMeasurementsKind, and links it back to the Job. 

Accessing the silhouettes

  • Use the S3Browser, and navigate to bvi-api-uk/SGB/
  • In there are the job folders (which correspond to the ID in the Job table), and inside each one are the two silhouettes.
  • Highlight the silhouettes, click Download and point them to a file
  • They will download (there is no indicator that they have downloaded!)
  • Opening these in Notepad++ show a list of co-ordinates, used by the image software to show the outline.
