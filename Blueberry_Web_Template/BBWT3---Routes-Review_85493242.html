Blueberry Group : BBWT3 - Routes Review

Created by Jason Cozza on Sep 21, 2020

Overview

Displays which groups have access to which pages of the web application.

  • Is presented in BB Grid Component format.
  • Header row.
  • Alternating shaded rows.
  • Ability to sort ascending/descending.
  • Filter search updates as you type.
  • Filter search allows for refined results through multiple search constraints.

Test Case

  1. Admin receives reports from users not being able to access a certain set of pages.
  2. Admin can impersonate a user to check.
  3. Admin sees that the user cannot access page due to lack of permissions.
  4. Admin can check via "View Route Access" and filter to the exact path.
  5. Admin can then make adjustments to page permissions to resolve issue.

For Testers

  1. Go to BBWT3 menu and click on "Security Admin".
  2. From the drop down menu click on "Routes Access".

For Project Managers

  • Be able to check prior to roll out which user groups have access to what.
  • Help identify restricted pages that are open, allowing you to lock down the web application sufficiently.
