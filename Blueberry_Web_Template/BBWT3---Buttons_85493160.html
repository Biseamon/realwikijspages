Blueberry Group : BBWT3 - Buttons

Created by Jason Cozza on Sep 21, 2020

Overview

On this page is an example of how a button is rendered with example code.


Within a form page, buttons should align with the left edge of the text.

Within a pop-up dialog that has labelled controls, buttons should align with the controls.

Within a pop-up dialog that contains no labelled controls, buttons should align centrally.

Example:

Example HTML:

<div class="ui-g">
    <div class="ui-g-12">
        <button class="ui-grid-row" type="button" pButton label="Button"></button>
        <button class="ui-grid-row ui-button-secondary" type="button" pButton label="Button"></button>
        <button class="ui-grid-row ui-button-success" type="button" pButton label="Button"></button>
        <button class="ui-grid-row ui-button-info" type="button" pButton label="Button"></button>
        <button class="ui-grid-row ui-button-warning" type="button" pButton label="Button"></button>
        <button class="ui-grid-row ui-button-danger" type="button" pButton label="Button"></button>
    </div>
</div>


Use "Save" as a button title on Edit pages, not "OK".

Use "Save" as a button title on popups that immediately commit data to the database, not "OK".

On tabs, place the Save button outside the tab control, and when Save is hit, validate all the tab pages and then save all the changes on the tab pages.

Button labels should be verbs or short verb phrases, e.g. "Register", but not "No Registration".
