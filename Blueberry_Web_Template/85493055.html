Blueberry Group : BBWT3 - Setting up a Local Email/SMTP 

Created by Jason Cozza on Sep 21, 2020

When developing your BBWT3 application, you may need to test out email functionality.

For security reasons, SES credentials should not be stored in the source repository for your project. Instead, credentials should only be stored with or retrieved by the application when it is running on a demo, test, or live system

For local development, use a local SMTP server which can capture your emails, allowing you to verify if your email functionality is working.

Windows

If you develop using Windows, use Papercut as a simple local SMTP server. Download it from the releases page.

Configure BBWT3 to use localhost as your mail server, and you should be good to go.

Docker

For a cross-platform solution, consider using Docker.

There are a few docker images out there which provide a painless way to set up an SMTP server.

The simplest solutions found so far are https://hub.docker.com/r/djfarrelly/maildev/ and https://hub.docker.com/r/mailhog/mailhog/.

Running Maildev

Configure BBWT3 to use 127.0.0.1:8025 as your SMTP address.

You can then view emails by browsing to http://127.0.0.1:8000

Running Mailhog

Configure BBWT3 to use 127.0.0.1:8025 as your SMTP address.

You can then view emails by browsing to http://127.0.0.1:8000
