Blueberry Group : BBWT3 - Single Sign On (SSO), SAML & OAuth

Created by Jason Cozza on Sep 21, 2020

Overview

Security Assertion Markup Language (SAML) is an open standard for logging into accounts, where authentication in one context has already taken place and another is requested elsewhere. This means that you are able to login with account "x" on system "y","z" etc. The single sign-on login streamlines the process of logging in by only requesting the user to sign in once. In this fashion it saves time and improves security; having one strong account that is remembered, instead of multiple accounts with weaker and more memorable passwords.

OAuth is an open standard for access delegation, commonly used as a way for Internet users to grant websites or applications access to their information on other websites but without giving them the passwords. This mechanism is used by companies such as Amazon, Google, Facebook, Microsoft and Twitter to permit the users to share information about their accounts with third party applications or websites.

SSO Support in BBWT3 (13-05-20)

Via OAuth

  • Google - Microsoft.AspNetCore.Authentication.Google
  • Facebook -  Microsoft.AspNetCore.Authentication.Facebook
  • LinkedIn - Microsoft.AspNetCore.Authentication.LinkedIn.

Configuration for these methods is via the Operation Admin → System Configuration screen in the BBWT3 UI.


Via SAML

  • Office 365 - Using the Sustainsys SAML 2 library - https://github.com/Sustainsys/Saml2
  • Active Directory / ADFS - Using the Sustainsys SAML 2 library - https://github.com/Sustainsys/Saml2

Configuration for these methods is via the Application Settings configuration file (appsettings.json)



Test Case

  1. A user accesses a remote application (BBWT3).
  2. The application identifies the user’s origin (such as from IP address).
  3. The user who is already logged in on one platform - the identity provider, tells the remote application that the user is authenticated. This is done in the form of a signed certificate containing the users email or username.
  4. The service provider (BBWT3) knowns the identity provider and validates the certificate.
  5. The user now has access to the service.

For Project Managers

  • Security Assertion Markup Language (SAML) is an open standard that helps keep accounts secure.
  • In addition to giving a smooth user experience with as little user engagement with the authentication as possible.
  • It allows a user who is signed in on one account to be automatically and seamlessly logged into another.
  • From the users point of view they are automatically logged in from only entering in one set of credentials. In reality there is work going on in the background server wise, which uses signed certificates to validate a user securely.
  • It is reusable. Once deployed it is a set and forget technology.
  • It doesn't use proprietary software with the added incurred costs associated with it.
  • Users love it – no longer do they need multiple logins and passwords for every service they use.
  • Supported Identity Providers (12/03/20): Office 365, Google. Some work has been done on Active Directory but this is not merged into BBWT3 code.



For Marketing

  • Ability to login with one account across multiple services (such as "login with Facebook").
  • A popular feature already in the mainstream.
  • User will not be put off by needing another username and password to remember.
  • Improves security by allowing the user to have 1 strongly protected account that can be used in multiple places.
