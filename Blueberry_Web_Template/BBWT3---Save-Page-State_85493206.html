Blueberry Group : BBWT3 - Save Page State

Created by Jason Cozza on Sep 21, 2020

Overview

Saves users filters on tables that would otherwise be reset if user leaves page for result and then returns.

Test Case

  1. User opens a page displaying a table.
  2. User then sorts or filters the table to findÂ pertinent information.
  3. User then opens an item on the table which redirects them to another page.
  4. Once user has finished and returns to the table, their previous filtering and sort settings persist.

For Testers

  1. From the BBWT3 menu click on "Operational Admin".
  2. Click on "Users".
  3. Change ascending order from one of the rows.
  4. Edit a user.
  5. Return back to "Users" page.
  6. The ascending column should still be ordered in the same configuration.

For Project Managers

  • Save page state refers to the ability of filtering and sorting a table, and having those settings saved and restored, after leaving the initial page. This inbuilt feature is one of the many things that BBWT3 implements correctly.
  • On sites that do not implement save page state, this causes frustration to the user. Especially in scenario's with large tables of data that need to be highly filtered in order to find the relevant information.
  • This feature of BBWT3 will be highly appreciated by the end user.
  • Undecided implementation: Currently being discussed whether to use localStorage or URL parameters (31-01-19).

For Marketing

  • User configured filters will persist when a user leaves a page and then returns to it after an action - saves users time and hassle by keeping filter preferences for later use.
