Blueberry Group : BBWT3 - Report a Problem

Created by Jason Cozza on Sep 21, 2020

Overview

BBWT3 has an inbuilt Report a Problem feature, that can take the users logged on account details (such as username and email address) and fill them into the template form.

  • Auto fills out user's details (however they still remain editable).
  • Client side validation (31/01/19)
    * Currently does not seem to have any (does not check if email address conforms to standard).
    * Unlimited char limit.
  • At request notifications can be setup to be sent via SMS.
  • Automatically generates a report within PTS.

Test Case

  1. User has an issue on the web application.
  2. User finds the "Report a Problem" page.
  3. Fills out form and sends to admin with severity.
  4. Admin is able to address the issue quickly so that other users do not face the same problem.

For Testers

  1. Go to the very bottom of the page, in the footer click on "Report a Problem".
  2. The username and email of the logged in account should automatically be filled in, enter the title of issue.
  3. Write the description of the issue in the "Problem Description" box.
  4. Select the severity of the issue encountered.
  5. Click Submit.

For Project Managers

  • Being able to have feedback on your web application can be useful for many reasons, but primarily for reporting back user issues.
  • BBWT3 includes an easily identifiable Report a Problem feature, so if a user has an issue they can quickly report it.
  • An Admin can set to who the email notification will be sent to.
  • By automatically filling out the users details this saves time and gives the system admins all the information that they need straight away.
  • If a test user is logged in, the details will be that which was entered when they entered their real details at session login. This way you can be sure to be able to contact the original tester if needed, instead of having to ask who was using the account at a specific time or day.
  • There is the option to have a title and a description box to outlie the issue encountered, what is more useful to you will be the ability of the user to state the severity of the issue. Ranging from minor cosmetic issue such as formatting or spelling mistakes, all the way up to complete system failure for multiple users.

For Marketing

  • Decreases the amount of effort a user needs to employ to report back issues with the service - by decreasing the amount of steps to provide feedback, leading to a better experience for all users.
  • Improved user engagement.
  • Improved communication between a user who spots the issue and the person responsible for fixing it - even on test accounts, because of the inclusion of entering real information once logged on.
  • Ability to focus efforts - by having the option for a user to state the level of severity of an issue, this allows for quick and easy prioritisation.
