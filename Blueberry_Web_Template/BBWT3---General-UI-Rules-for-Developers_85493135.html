Blueberry Group : BBWT3 - General UI Rules for Developers

Created by Jason Cozza on Sep 21, 2020

Overview

This page gives a quick recap on the general rules that should be used when coding for BBWT3.


  • Always use "email" rather than "e-mail".
  • Use the following capitalisation rules:
    • Title case for page titles, tab page titles, headings,titles of popup dialogs, titles of popup messages, menu labels, titles of grid columns, titles of table columns, button labels. Use http://individed.com/code/to-title-case/ for determining the proper English title case - it's not as simple as capitalising every word. We will be rolling out a tool to help developers with this sometime soon.
    • Sentence case for; labels, checkboxes, drop down menus, lists, radio buttons, text edits, labels for groups of radio buttons and checkboxes, strings in drop down menus and lists, text inside popup messages, text inside progress indicators, links to other pages. Sentence case is the 'normal' use in English prose: only the first word is capitalised, except for proper nouns, acronyms, etc. See http://en.wikipedia.org/wiki/Letter_case for details.
  • We should never use "Click _here_" - we should always use more natural sentences.
  • Use one column layouts; avoid two-column layouts.
