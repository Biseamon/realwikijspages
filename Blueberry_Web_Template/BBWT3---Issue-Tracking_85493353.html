1Front-end request optimize04/18

Try to optimize requests to server that happen on each page load. This sometimes may delay main business request. Examples:

  • We load menu items from server on each page open, can be cached
  • Statistic requests like a route loading time and also Feedback feature's request should be queued last

So the idea is not to give a chance for other requests to delay user from viewing of business data

2DbDoc metadata "Required" validation rule04/19

"Required" validation rule conflicts with model not-null rule. When we add such rule it visually seems added but after reloading page it won't appear.

3Getting rid of application reloadings07/05Rework user.service.ts and other related things to rid from redundant application reloadings. Namely: Log-in, Log-out, User's avatar change.
4Grid's export feature07/05Rework exporting feature of Grid. Remove all code on back end which relates to exporting. All logic of "xlsx" and "csv" exporting should be on front end. There are lot of libraries and native tools of JavaScript to encapsulate such function within front end inside Grid.
5Async validators07/05We need to include async validator to our solution. It will help users to see their input errors immediately without request sending. Next pages required such behaviour: Registration (email), User Editing (email), Profile (email), Static Pages (alias), perhaps somewhere else. For such field should be visual loader in the right side which indicates background validation process.
6Grid rows selection by row clicking07/05

I often hear desire about Grid rows selection by just tapping on row. We agreed about special column with checkboxes and radio buttons but seems that this way is not suitable for all customers. Need to provide new selection mode which allow to select rows by tapping.


(Requested by David Foster (not only) 07/04)

7Visual indicators for pending07/05

Provide visual loader to inform user about background process instead of "dead" page. Such loader might be placed:

1) on submit button when we sending form
2) on the right side of inputs when async validation in process
3) on page instead of form (or on all form inputs and submit button at the same time) when we need to load some data on form before let user edit this
4) instead of image when it loading

8Visual loader for app09/07After removing default theme (Indigo) which was written by default in index.html we need some neutral visual loader when application loads.
9Companies logo uploading30/07

Presently is possible situation when we can upload some file but not press the "Save" button then. In a result we are getting an unrelated "dead" file which stored in file system but never used. Also we can change the logo but old image would not be deleted.

If talking about this situation more globally, FileSorageController is redundant because files uploading in pure form never make sense. That process always performs along some business-action. By this cause we should use FileStorageService inside controllers which allow files uploading, instead of separate FileStorageController.
