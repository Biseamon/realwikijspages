Blueberry Group : Kendo UI Details

Created by Jason Cozza, last modified on Sep 21, 2020


The good thing about kendo widgets is that they are completely styleable, while many native html controls are not. One of such a cases is dropdown list (select tag). But the problem is that migrating to kendo dropdown has many obstackles which are very hard to discover. And can lead to rejection of the kendo with very unpleasant consequences. Here are the problems I solved in order to be able to use it:

Kendo drop-down list control

  • use kendo.data.ObservableArray as datasource, otherwise you will see your data sporadically. And do not reassign it (otherwise the binding gets lost) - use 'splice' to apply any changes.
  • use k-option-label like “‘label’” - two quotes, otherwise it doesn’t work
  • use input tag instead of select (can't remember why)
  • to hide placeholder from the dropdown list use k-on-open=”onDdOpen(kendoEvent) . Define the handler in root scope like:$scope['onDdOpen'] = (event) => {               $(event.sender.ul[0]).addClass('hide-default-option');           }Then use css:ul.hide-default-option{   li:first-child {       display: none;
       }

    }

    This is very useful technique in general to style any kendo widgets which use animation containers (dropdown, menu, popup window)


As result we have html for dropdown like this:

<input kendo-drop-down-list
                                   k-data-text-field="'CompanyName'"
                                   k-data-value-field="'Id'"
                                   k-data-source="viewModel.Clients"
                                   ng-model="viewModel.SelectedClientId"
                                   k-option-label="'select a value'"
                                   class="form-control"
                                   k-on-open="onDdOpen(kendoEvent)">
 </input>


It's worth to mention that any angularjs binding should contain ".", otherwise bindings may stop working in some cases. It's well know rule.

General notes

Generally, kendo data binding is not compatible with angularjs bindings (http://kendo-labs.github.io/angular-kendo/#/datasource). It's better to use kendo widgets from the beginning, because the migration is not easy and it leads to wasting a lot of time: first time you use native html controls and spend a lot of time debugging and fixing various bugs most of them related to angularjs bindings; then you discover that you need styles and kendo and fix similar problems again. I've used the angularjs in second project already, and kendo data binding and MVVM implementation looks much more reliable and much less error prone than angularjs data binding.
