Blueberry Group : Important note to Add a background Image on Reports (SSRS)

Created by Jason Cozza on Sep 21, 2020


I noticed that SQL server reporting services is rendering the images in a different way for the reporting viewer control and the PDF renderer.

So the workaround I found is to create 2 templates, I mean 2 .pgn files with the graphics called VerticalImagePDF.png and VerticalImageScreen.png and to add some code that dynamically use both graphics depending of the renderer that is being used , later I uploaded both images to the report server ( you can find those files under the path “/Report Parts/”) the idea is to reuse those images on all reports

Important Notes:

– It is important to use external images because it will allow you to reuse the same template (pgn file used as background) for several reports this will allow you make your life easier maintaining the background graphic on reports, please don’t use embedded images because if someone ask you to modify templates then you will need to edit every image on all reports separately.

– Every template should be modified by making the image as background image to the body of the report and adjusting its size to match with the body (if you don’t listen and do something different then you will see the image repeated several times on the same page) — You will need to perform different tests for the template that will be used for the Report viewer control (I mean the one that will be showed on the web application) and the template that will be used for the PDF renderer.

Please see below more details about how to add code that dynamically use both graphics depending of the renderer  :

1.- Click on Report Body properties and click on Fx button to add an expression

2.- Add the expression code and change the template names to suit your needs:

=IIF(Globals!RenderFormat.Name = “PDF”, “/Report Parts/VerticalImagePDF.png”,”/Report Parts/VerticalImageScreen.png”)

 

Note: The code above is using the variable Globals!RenderFormat.Name to determine what renderer is being used at runtime, so if the PDF renderer is being used then it will return the string “/Report Parts/VerticalImagePDF.png” otherwhise “/Report Parts/VerticalImageScreen.png”
