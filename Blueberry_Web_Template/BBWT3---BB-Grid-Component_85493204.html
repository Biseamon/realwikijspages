Blueberry Group : BBWT3 - BB Grid Component

Created by Jason Cozza on Sep 21, 2020


Merger (31-01-19)

Filter and grid will soon be merged.


Overview

In modern business applications, the display of data in a table or grid is a very common requirement, BBWT3 fulfils this requirement by building on from PrimeNG. In this heavily modified version, it allows for the inclusion of more customised UI elements such as custom buttons. It is also built for dealing with larger and complex data sets, and allows for editing data inline without the need of a "save" button.


  • Contains header row for columns at top.

  • Highlights active sort order in header row.
  • Highlights which direction the sort is in by blanking out other arrow.

  • Alternating row shading.

  • Ability to delete individual rows.

  • Ability to delete whole database.

Developer Notes

There are 3 different types of grid; Page editing, Inline editing, Popup editing.

Choose one of these types and use selected type code in your grid. In grid.component.html you can find comments about how grid parameters can be used.

Column Sizing

  • Column size should be defined in characters since it is straightforward to identify how many characters are required. It is not easy to correctly define this in pixels.
  • You need to add validation to ensure that: minimum <= maximum, etc.


Pseudocode

# This table can contain fixed width columns (MIN equals MAX) and variable width columns (MAX is larger than MIN)
# Calculate the maximum width of the table without requiring a horizontal scrollbar
Calculate maximum desired table width = $TOTAL_MAX_WIDTH
# Identify whether we have space to support minimum widths:
Calculate the sum of all minimum table widths = $TOTAL_MIN_WIDTH
IF $TOTAL_MAX_WIDTH =< $TOTAL_MIN_WIDTH THEN {
       # Use the minimum specified widths for all columns.
       # this might result in a horizonal scrollbar.
}
ELSE IF $TOTAL_MAX_WIDTH > $TOTAL_MIN_WIDTH THEN {
       # We have some room for manoeuvre, good.
       # Now allocate remaining space based on the maximum specified column width for each column.      
       # Calculate available width as delta between maximum allowed and minimum required.
       $TOTAL_AVAILABLE_WIDTH = $TOTAL_MAX_WIDTH - $TOTAL_MIN_WIDTH 
       # 1. Use the specified maximum for each column to calculate a ratio expressed as a percentage, eg:
       # Column 1 = 5%
       # Column 2 = 25%
       # Column 3 = 25%
       # Column 4 = 15%
       # Column 5 = 20%
       # Column 6 = 10%
       # If done correctly the total will always be 100%
       Sum of maximum width for all variable width columns = $TOTAL_WIDTH_FOR_ALL_VCOLUMNS
       FOR EACH VARIABLE WIDTH COLUMN{
              # We don't actually need to convert this to a percentage in the final implementation
              $COL_PERCENTAGE = ((100 / $TOTAL_WIDTH_FOR_ALL_VCOLUMNS) * $MAX_WIDTH_SPECIFIED_FOR_THIS_COLUMN)
              $COLUMN_WIDTH = ($MIN_WIDTH_SPECIFIED_FOR_THIS_COLUMN + ($COL_PERCENTAGE * ($TOTAL_WIDTH_FOR_ALL_VCOLUMNS / 100)))
              # But we can't exceed the maximum specified width for this column
              IF $COLUMN_WIDTH > $MAX_WIDTH_SPECIFIED_FOR_THIS_COLUMN THEN{
                     $COLUMN_WIDTH = $MAX_WIDTH_SPECIFIED_FOR_THIS_COLUMN
              }
       }
       FOR EACH FIXED WIDTH COLUMN{
              $COLUMN_WIDTH = $MIN_WIDTH_SPECIFIED_FOR_THIS_COLUMN
       }
}

  1. User in browser goes from full screen to having the browser on half of the screen.
  2. User experience is maintained as grids resizes automatically.

For Project Managers

Delivering a powerful and polished grid UI goes a long way to making applications easier to use for novices, and more powerful for advanced users. Blueberry has worked to customize the PrimeNG grid to:

  • Provide multiple ways to support master-detail editing of records within the grid.

  • Support advanced filtering of grid records.

  • Add built-in Export capabilities.

  • Add automatic grid-column sizing, and improve the Window-Size responsiveness of grid-column sizing.

  • Bug: After editing user / clicking "Save" Does not redirect back to list view (31-01-19).

For Marketing

  • Have tables automatically resize depending on device being used - mobile, tablet or desktop.
