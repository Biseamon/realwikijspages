Blueberry Group : BBWT3 - Feedbacks

Created by Jason Cozza on Sep 21, 2020

Overview

This feature allows for a user to report an issue.  This component saves tester's time by automatically creating a ticket/issue in the project management tool, which a project manager or tester can then track. 

  • User is able to place a "pin" on the screen where an issue is located.
  • Has the ability for a user to upload a description of a problem using a form with a title and comment box.
  • When the pin is moved a screenshot is taken automatically on the server side and sent with comment to developer.
  • User has ability to use speech recognition.
  • Has the ability for the user to describe the problem with voice, and see their speech displayed in the feedback box in real time.
  • User is able to click on the screen where the problem is located.
  • Has the ability for real time communication when multiple users are on the same page thanks to websockets.

Test Case

  1. User has an issue with the web application.
  2. User is able to take a screenshot of the issue and upload it directly without any 3rd party software.
  3. User has the ability to use their voice and have it turned into text via an API instead of typing.
  4. Admin receives the message and is able to investigate and fix the issue quickly.

For Testers

1a. For full screen mode click on the talking bubble with the explanation mark in the centre.

1b. For tablet/mobile mode click on the 3 horizontal bars and click on "Feedbacks".

2. A new windows will open with automatically selected filters.

3. Click on "New" in this new window.

4. On the main web application page enter in a title and description.

5. If you wish you can accept the voice recording function and have your voice be the input.

6. Click on "Save" to submit your issue.

For Project Managers

  • Quick and easy method of reporting issues.
  • Ability to describe the issue in detail due to the various media options available.
  • Can demonstrate issue in real time.

For Marketing

  • Allows users to send issues to admin quickly.
  • Able to upload a screenshot of the issue without the use of 3rd party software.
  • Able to make notes through voice recognition.
  • Real time communication of issue can be achieved if more than one party is on the same page as the issue is being made.
