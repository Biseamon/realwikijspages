Blueberry Group : Reporting Details

Created by Jason Cozza on Sep 21, 2020

This report receives the parameters below which in turn receive an array of values for each parameter:

 

 

All parameters are passed to the stored procedure named sp_getGroupReportData at the very beginning  I used dynamic SQL (Ad-hoc queries) by using within the SSRS report a dataset with Query Type = Text,  but later I changed my mind and changed the report logic to use Stored Procedures  because of once the report was deployed ,  the customer asked for more changes making the  query more complex and finally SQL Server complained about this, After googling a bit I found out that SQL Server query optimizer it doesn't like complex queries which in turn receives too many parameters as  array of values.

 

The workaround to fix this issue  was to use temporary table to process the big query into several steps by using temporary tables

See below how the online version of this report look like, Additionally you can see attached the Excel/PDF version of this report along with the RDL files named GroupTrainingReport.rdl, GroupTrainingReport_EXCEL.rdl , GroupTrainingReport_PDF.rdll, you will also will find out the file named TSQL_GroupReport.SQL with the stored procedures required to be able to execute this report
