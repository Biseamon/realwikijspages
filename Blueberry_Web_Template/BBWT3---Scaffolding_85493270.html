Blueberry Group : BBWT3 - Scaffolding

Created by Jason Cozza on Sep 21, 2020

Overview

Scaffolding is done through dotnet, it allows for the ability to select which modules are wanted to be included/excluded in a new project.

Currently this feature is not implemented due to picking and choosing which elements to include is difficult with the current architecture of BBWT3.



(31-01-19)

Currently the best approach to starting a new project using BBWT3, is to simply Fork from the current state of BBWT3.


Advantages

  • Smaller app - no added bloat from having features that are unnecessary.
  • Less dependencies -  because we are using only what we need.
  • Faster more responsive app.
  • Easier to debug.

The code BBWT3 codebase has modules located in "bbwt3/modules". These are a mixture of third party modules and Blueberry code. When adding third party libraries to a forked BBWT3 project, they must not be put in "bbwt3/modules" they must be kept separate. We recommend "bbwt3/thirdparty".

Why? This is to help with code audits and specifically to allow automated tools such as SonarQube to be implemented correctly.
