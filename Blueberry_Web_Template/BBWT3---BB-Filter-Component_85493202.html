Overview

The filter in BBWT3 is built on PrimeNG GUI framework but has been expanded upon to meet the needs of large data sets.  It is able to rank data from minimum to maximum values, as well as contextual search. The search parameters that can be used are:

  • Ability to search by "Greater than (x)".

  • Ability to search by "Contains (x)".

  • Ability to search by "Ends with (x)".

  • Ability to search by "Length equals to (x)".

  • Ability to search by "Exact name (x)"

Test Case

  1. User has to search through employee's but only has partial information.
  2. Using the advanced filters user is able to find the specific employee from database.

For Testers

  1. From the BBWT3 menu click on "Feature Demo Pages".
  2. Click on  "Employees (0Data)".
  3. It will display all data inputted, you can now sort and search through the data.

For Project Managers

  • With BBWT3 you can set tables up to list data with varying parameters.
  • You can have a search field that will search entire strings in the database for "containing".
  • Useful in situations where you are unsure of an entire field but remember part of it. The end user would benefit greatly from this feature with web applications dealing with many fields containing multiple clients.
  • BBWT3's implementation of PrimeNG is suited for large data sets.

For Marketing

  • Powerful inbuilt search features, help you to identify the information needed.
