Custom HTML elements

Angular-kendo templates won’t work with custom html elements like <pts-entity-lookup>… </pts-entity-lookup>) but will work with attributes like <div pts-entity-lookup>…</div>.

Entity framework

Entity Framework throws an exception if you try to select a result into a class which has database mapping. So more classes are needed just to select results, because anonymous classes can’t be used with AutoMapper. 

Calling stored procedures is inconvenient in case of Fluent API use: http://www.c-sharpcorner.com/UploadFile/ff2f08/call-store-procedure-from-entity-framework/

Cross-domain

Custom header Access-Control-Allow-Credentials must be set in configuration together  with using Microsoft.AspNet.Cors package from NuGet. More on CORS in Asp.NET here http://msdn.microsoft.com/en-us/magazine/dn532203.aspx. jQuery doesn’t send cookies for cross-domain requests by default, so one need to set ‘withCredentials’ flag for xhrFields manually (appropriate section here http://api.jquery.com/jQuery.ajax/).

Changing ODATA routing

Use WebAPI 2.2  http://www.asp.net/web-api/overview/releases/whats-new-in-aspnet-web-api-22. New OData doesn’t support DateTime structure (http://aspnetwebstack.codeplex.com/workitem/1753) and one needs to convert it to DateTimeOffset.

Add a new menu item

1. Make changes in app.ts
2. Make changes in Configuration.cs
3. Log in as an admin, go to Admin -> System Configuration -> Menu Settings, press Reset button.

To configure debugging *.ts files

1. Set IE as a default browser
2. Open the browser, go to Tools -> Internet Options -> General -> Check ‘Delete browsing history on exit’ option, then go to ‘Browsing history’ -> Settings -> Check for newer versions of stored pages – Every time I visit the webpage’.

Problems with LESS in VS 2012

When you working with LESS in Visual Studio 2012 you may faced with compiler error of * .less files – “less: Unrecognised input” (see. more http://stackoverflow.com/questions/18180160/less-unrecognised-input). In addition, we observed problems at compile, unless we explicitly specify the encoding. To fix this, do the following:

1. Make sure you have the latest updates Web Essentials 2012: http://vswebessentials.com/download

2. Added in headers on every * .less file the next code:

#fix {}
charset “utf-8”;

In VS 2012 you should also make next:

1. In VS menu – Tools -> Options -> WebEssentials -> LESS -> Compile on Build = true

2. Restart Visual Studio

Sql server reporting services (SSRS) IE bug..

Long running reports are not well rendered in IE (Any version) because a lot of white space is rendered at the top, but they works fine on Chrome and Firefox to duplicate this issue just create a new report, add a label  followed by a tablix linked to a dataset that retrieve more than 2000 records.

Sql server reporting services (SSRS) Sub-reports

For long running reports the  subreports approach should be avoided, the main reason is that SSRS open a new database connection per every subreport and submit a new query. So If the main report iterate over 1000 records and the sub report have 2 dataset then SSRS will submit 1 query for the main report plus 2000 queries for the subreport

Migrations

For BBWT2 projects we typically use “migrations” to keep the C# model and the SQL Server schema co-ordinated. This can be done manually or automatically.

We can lose useful development time when we use manual migrations  - because they are time consuming to write. But we can also lose useful development time when we use automatic migrations – because there are merge issues when multiple people are committing schema changes. Please read https://msdn.microsoft.com/en-us/data/dn481501.aspx . It details the merge problems and how to quickly work around them.
