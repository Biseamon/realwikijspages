What support do we have for logging?

We log both client-side and server-side. We send the logs server side and use log4net as a log events processor.

Log4net can persist its logging information to any of a wide range of options including files, databases, or email.

But by default it logs only to a network port. Developers are intended to RDP to the server and use Log2Console or Sentinel to view the logging information.

To configure different targets use the log4net web.config section. The file writer configuration is there already but commented out. You can see configuration examples for standard log4net listeners here: http://logging.apache.org/log4net/release/config-examples.html

A short list of available targets is here: http://logging.apache.org/log4net/release/features.html

By default, for a live environment, you should use file-based log storage and a logging level of "error” or “fail” – if you log all trace messages you will soon run out of file space. Log2Console should not be run continuously in the live environment either - it has been observed using a Gigabyte of memory.
