Blueberry Group : BBWT3 - Allowed IP

Created by Jason Cozza on Sep 21, 2020

Overview

The Allowed IP setting is a feature to restrict account access. It specifies a whitelist of specific or ranges of IP addresses that have permission to access an account - such as the admin account.

  • Only allow specific IP addresses to access certain accounts.

Test Case

  1. Admin has disallowed login to admin accounts from anywhere outside of local network.
  2. An attacker is unable to login to an admin account even with full credentials due to not being on the IP whitelist.
  3. An off-site admin wishes to login from remote location.
  4. First the off-site admin has to VPN into the company network.
  5. Off-site admin can now login to admin account.

For Testers

  1. Go into "Operational Admin" from the BBWT3 menu.
  2. Click on "System Configuration".
  3. Go to the "Allowed IP" tab.
  4. Click "Add".
  5. A new windows will be displayed, and you will be prompted to enter in the allowed IP range.
  6. Once correctly entered, click on the "Save" button.
  7. Finally click on "Save" at the bottom of the page again to save all changes.

For Project Managers

  • BBWT3 has strong security built into its features.
  • You can have a whitelist of IP addresses which can only access a certain account. For example an admin account may be protected by only allowing business IP addresses to log into it.
  • Improving security by blanket denying any access outside of your own internal network.
  • Limitation: Potential issues with remote management. However this can be negated by requiring an admin to first connect to the business VPN before logging into the account.

For Marketing

  • Improved security - by only allowing certain IP's to be able to log into accounts enhances security through the use of a whitelist.
