Overview

2FA (Two Factor Authentication) utilises one-time only passwords in conjunction with the user's standard username and password to login .  This can be achieved via software ie. TOTP (Time-based One-Time Password) authenticator app (such as Google Authenticator), or via hardware (U2F). It improves security by adding an additional layer to the verification process to prevent unauthorised access.

User logs in with username and password, then is prompted to complete two factor authentication with the method that has been set up.

  • If the 2FA device is no longer accessible, 2FA can be removed through the use of a recovery code.

 

Limitations

U2F is currently disabled when BBWT3 is installed on a Linux server, because the U2F server side library is not working yet on Linux because ECDsaCng is not supported.

Test Case

  1. Admin user receives malicious email with a link to a phishing website.
  2. User follows link and attempts to use login credentials.
  3. Attacker now has both username and password of the admin user.
  4. Attacker attempts to login to real site with stolen credentials.
  5. Attacker is unable to successfully login due to not having the one time only password, generated by two factor authentication.

For Project Managers

  • Two factor authentication provides additional security on accounts by utilising one time only passwords in conjunction with a username and password.
  • 2FA helps prevent phishing attacks from being successful as even if a users account and password are compromised, this information is useless to an attacker as they do not have physical access to the TOTP device associated with the account.
  • Alternately the use of Universal 2nd Factor (U2F) physical keys such as FIDO can be used instead as the two step authentication. They work in a similar manner to Google Authenticator but have a physical button to be pressed at time of login.
  • The enhanced security features will make accounts more protected and thus the business as a whole.
  • However users must still be educated to be security conscious. Inserting unverified USB keys into work computers still poses a significant security risk.
  • Both of these methods introduce a further step that the user must complete before being able to access their account. This may initially cause confusion with end users, but they will soon become used to the additional step.
  • Limitation: The U2F method currently is not supported if BBWT3 is installed on a Linux server.

For Marketing

  • Uses modern security practices - one time only passwords.
  • Emerging popularity in the mainstream for such technology.
  • Highly resilient to phishing attacks - even if an attacker has a username and password, without the code from the Authenticator app or the physical hardware USB (e.g FIDO), they cannot access the account.
