Blueberry Group : BBWT3 - Page Editor

Created by Jason Cozza on Sep 21, 2020

Overview

Custom applications need ways to guide end-users. However, writing a manual or online help is a significant investment – and if an application is changing rapidly, writing the help can be a never-ending task.

 Blueberry’s view is that help should be delivered in two ways:

  • As tooltips within the application, which give users immediate guidance and examples of what data is expected in a control.
  • As a separate help file which is organised by task or process – so “How to Create a User”.

We see less value in creating screen-by-screen help pages – well, until it can be automatically generated.

As a first step in automating system help documentation, BBWT3 includes an interactive tooltip feature:

  • Developers include (i) icons next to controls.
  • An administrator with the right permission can edit the text for these tooltips interactively directly within the application.
  • The text of the tooltips is automatically sent back to be included in system source code.

Why is this useful? Tooltip text generally needs to be specified by the customer, because it is the customer’s system and they have the domain knowledge.  In previous systems, customers would have to go through and write a long list of the tooltip texts and send to us to include. This was a time-consuming process. By automating this process we save a lot of time, and make it easier to ensure that we have good, strong tooltips in place.
Tooltips are being replaced by a HTML editor, where you can enter a basic HTML editor and create or edit the text for prompts or tooltips.

Test Case

  1. The web application has been fully developed and is online for client.
  2. Client admin gets feedback about web application from users.
  3. Users need clarification on a feature.
  4. Admin can create a tooltip icon and description next to the feature.
  5. Client does not need to spend time and money on developers to do this.

For Testers

  1. From the BBWT3 menu click on "Operational Admin".
  2. From the drop down menu click on "Menu Configuration".
  3. Under "Footer Menu" in the grid click on the "i" icon next to "Order".

For Project Managers

  • Ability to have notes on features.
  • Have relevant information available on screen right next to a feature or section.
  • Empowers users.
  • Admins can make changes and use custom text without the need to contact a developer.

For Marketing

  • Help icon's to give out extra information on a feature that a user might not know how to use.
