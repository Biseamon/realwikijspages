Blueberry Group : BBWT3 - Cache Testing

Created by Jason Cozza on Sep 21, 2020

Overview

This feature demonstrates caching in BBWT3.

  • All elements are uniquely named.
  • When an update or change occurs, the original file name is replaced with a new named one.
  • Because the new element is differently named, the users web browser downloads changed/update version.

Test Case

  1. User is on web application.
  2. Admin updates web application by replacing existing elements.
  3. User on next page action will see changes.
  4. Â User's browser does not retrieve old files from cache, as browser sees changes as new files due to them being uniquely named.

For Testers

  1. From the BBWT3 menu click on "Technical Admin".
  2. From the drop down menu click on "Cache Test".
  3. Displayed will be a current number (e.g "1").
  4. Click on "Increase".
  5. Refresh the page.
  6. Number should have increased to reflect the amount of times "Increase" was clicked.

For Project Managers

  • Ability to make small changes to images or scripts and have them instantly be viewable.
  • Having uniquely named files means that there won't be issues with cacheing in browsers preventing users from seeing the latest changes.

For Marketing

  • Ability to make changes no matter how small or regularly, and have them instantly viewable by your users, through uniquely naming each element; be it image, script or media files.
