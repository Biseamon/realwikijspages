Blueberry Group : BBWT3 - Real User Names

Created by Jason Cozza on Sep 21, 2020

Overview

When a tester or BB person logs into a BBWT3 test system, they often use a test account. For some future features, we need to know the real name / ID of the current user.

  • Upon successful login with a test/demo account user will be prompted to input real info.

  • This information will be called upon when Reporting a Problem.

Test Case

  1. A tester account can have the role “BBTester”:


  2. When the tester logs in for the first time, we ask to fill out with their real data:


  3. This data is used when a tester reports a problem:


  4. The form is filled with real data. We can change this data in the user’s profile.

For Project Managers

  • Be able to track down testers.
  • When a single account is used by multiple testers trying to find the one who reported an issue could be complex.
  • BBWT3 forces users on test accounts to fill out their real contact information.
  • However a tester must be made aware of the reasons behind this otherwise they may still fill out the details with unidentifiable information.
  • Limitation: Does not check for validation when entering in information (such as checking email address is in valid format).

For Marketing

  • If you have test accounts for multiple users tracking down who reported what can be difficult - with BBWT3 when a new test users logs into the account they will be prompted to enter their real details, resolving this issue.
