Blueberry Group : BBWT3 - Page Load Timer

Created by Jason Cozza on Sep 21, 2020

Overview

Page load timer is a feature that displays how long it took in seconds to load a particular page. It displays this value in the top right corner when enabled.

  • When enabled displays stopwatch icon, with the load time in seconds for currently displayed page.

Test Case

  1. Prior to deployment of web app an admin may wish to check for consistency across the site.
  2. Admin wishes for all pages to be responsive for the user.
  3. Admin goes through links of web app and reviews loading time.
  4. Any pages with longer load times can be tracked down and investigated.
  5. Admin can then make changes such as resizing images.

For Project Managers

  • Having page loading times easily accessible gives a quantifiable value of site responsiveness.
  • You will be able to identify any problem pages, and then be able to attempt to fix issues, to make the user experience more responsive.
  • Having the ability to store all load times within a database that you are able to search and sort through, helps in identifying problems.
  • There is a log of the user and path to which the site was loading on display within the database. This way you can identify who had the issue, as well as what page they were attempting to access.
  • Limitation: Only provides basic search and filtering capability (31-01-19).
  • Limitation: Does not update the load time when filtering search or sorting tables (31-01-19).

For Marketing

  • Able to search for problem pages prior to roll out - enabling a smooth user experience throughout the site.
  • Able to identify through logs with who and where an issue occurred.
