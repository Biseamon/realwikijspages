Blueberry Group : BBWT3 - Validations

Created by Jason Cozza on Sep 21, 2020

Overview

BBWT3 does validation across all fields and user data entry, to ensure security of the app by sanitising input data, and validity of the supplied information.

  • Can store data as:
    • Nvarchar.
    • Varchar.
    • Char.
    • Bit.
    • Double.
    • Float.
    • Integer.
    • Decimal.

Test Case

  1. User is entering data into a database field.
  2. User attempts to submit.
  3. Data is checked and an error message is returned.
  4. Alerts user to have entered in the data into wrong column.

For Testers

  1. Go to BBWT3 home page.
  2. Click on "then sign up".
  3. Enter email address.
  4. If email address is not in the correct format it will not allow for user to continue.

For Project Managers:

  • Can store and accept data in a plethora of ways.
  • Data can be screened for validation within select fields or forms - such as ensuring that an email address contains an "@" as well as a "." and in the right order.
  • Ability to set character limits.
  • Has the flexibility to accept or reject null values.
  • Can check for duplicate entries.
  • Sanitises input data to prevent attack.

For Marketing

  • Ability to store data in a multitude of formats.
  • Sanitises form input data to prevent attack.
  • Ensure your users are entering in valid data.
