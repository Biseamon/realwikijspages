Blueberry Group : BBWT3 - User Lock Feature

Created by Jason Cozza on Sep 21, 2020

Overview

Users can be locked out through the use of the "Failed Attempts at Password Entry" setting.

If a user incorrectly enters a password the system is able to lock the account depending on the settings that an admin has chosen. For example they are able to lock accounts after “n” number of consecutive failed attempts within a given number of minutes.

  • Prevents brute force attacks.
  • Times out accounts based on settings.
  • Can lock accounts that have had too many failed attempts - requiring an admin to manually unlock them.

Test Case

  1. Attacker who knows a valid admin username attempts to brute force password.
  2. Admin has configured "Failed Attempts at Password Entry" to lock user out until manually reset by another admin.
  3. Attacker is now unable to attempt any other passwords on that account.
  4. Admin will be notified that an account has been locked.
  5. Admin will have to manually unlock the account, and has become aware of an attempted breach as detailed within logs.

For Testers

  1. From the BBWT3 menu click on the "Operational Admin" section.
  2. From the drop down menu click on "System Configuration".
  3. Go down to "Failed Attempts at Password Entry".
  4. Click on the box next to "Users are locked out of their accounts after".
  5. Enter in how many consecutive failed attempts at login that will trigger the lock.
  6. Enter in how many minutes to count the failed logins for.
  7. Select from the box either; how long the account will be locked for, OR if the user will require manual admin changes to unlock their account.

For Project Managers

  • Having timeout and lock account features can help prevent accounts being compromised.
  • A brute force attack could be utilised where a login name is known, such as "admin", but the password is not.
  • The attacker will attempt many different passwords to try and get the correct one, and gain unauthorised access to an account, potentially causing serious harm and disruption.
  • By using the user lock feature, you have the ability to deny access to an account that has had too many failed attempts at login, either for a set amount of time, or permanently until it is manually reviewed by an admin.
  • Issue: If user locks account, can be unlocked by doing password reset without admin intervention.(31-01-19 #177006).

For Marketing

  • Inbuilt brute force protection - accounts that have had too many failed login's are automatically timed our, or if necessary blocked until an admin has unblocked them manually.
