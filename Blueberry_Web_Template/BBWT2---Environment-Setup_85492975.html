Note about difference between 2008 and 2012 server setup:

The latest release of TS can be used on 2008 is 0.9. Windows 2012 server can use any release where version number is >=1.0.

1.0 sources are compatible with 0.9 (but 0.9 not always compatible with >1.0) - please do development using TS version >1.0

Database setup

The application manages the database structure and creates the initial set of data records itself - you don’t need to use any separate tool for this. But you still need to create an empty MSSQL database and modify connection string in connections.config file. Rename or copy connections.config.template file. That file is an example of a correct configuration file.

Make sure that the IIS user has enough rights to be able to create/update database structure. DB_OWNER is the best role to assign to IIS user. Use SQL Server Management Studio to do this.

Nuget setup

Go to Tools/Nuget Package Manager/Package Manager Settings and turn on ALL checkboxes in “package restore” section

Log viewer (use it to debug BBWT2 application)

BBWT2 has a built-in log system based on the NLog logger. Both server and client side log messages to a single log destination. It is possible to configure NLog to use a file but for BBWT2 we a NLogViewer UDP listener.

I recommend to use Sentinel log viewer http://sentinel.codeplex.com/

Run the tool application and create new listener. Add NLog on provider registration step with default options value.



As soon as the listener is created you will be able to see log messages in the Sentinel application window.



Another possible utility to use is “Log2Console” which may work even better
http://Log2console.codeplex.com
Just run the utility and see log messages - no any configuration required.
